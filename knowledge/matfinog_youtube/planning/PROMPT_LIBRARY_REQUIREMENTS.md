# Prompt Library Requirements

This document translates the Workflow Browser Design into specific, actionable requirements for future implementation.

## REQ-001: Browse Prompts by Workflow
- **Description:** The system must allow users to view prompts grouped by their `source_workflow_id`.
- **Input Artifacts:** `prompt_library.jsonl`, `prompt_library_index.csv`.
- **Output Behavior:** Display a list of the 8 canonical workflows and allow the user to select one to see all associated prompts.
- **Safety Rule:** No automated summarization or synthesis of the workflow's overall strategy is permitted.
- **Acceptance Criteria:** A user can select "Risk First Strategy Review Workflow" and see exactly 59 prompts (or the number defined in the index).
- **Source Grounding:** Based on the `workflow_taxonomy.yaml`.

## REQ-002: Browse Prompts by Topic
- **Description:** The system must allow users to view prompts grouped by `source_topic_ids`.
- **Input Artifacts:** `prompt_library.jsonl`.
- **Output Behavior:** Display a list of topics (e.g., `risk_management`) and show prompts that contain that topic in their `source_topic_ids` array.
- **Safety Rule:** Topic groupings must not be used to generate cross-topic trading signals.
- **Acceptance Criteria:** A user can select "Execution and Microstructure" and see all prompts related to slippage and fill rates.
- **Source Grounding:** Based on the `topic_taxonomy.yaml`.

## REQ-003: Filter by Prompt Type
- **Description:** The system must allow filtering prompts by `prompt_type`.
- **Input Artifacts:** `prompt_library.jsonl`.
- **Output Behavior:** Provide a filter control for types like `validation_question`, `risk_review_question`, `research_question`, etc.
- **Safety Rule:** No new prompt types can be dynamically generated by the UI.
- **Acceptance Criteria:** A user can filter the library to show only `validation_question` prompts.
- **Source Grounding:** Sourced directly from the pre-processed JSONL.

## REQ-004: Show Source Video Traceability
- **Description:** Every prompt must display a direct link to its source video.
- **Input Artifacts:** `prompt_library.jsonl`.
- **Output Behavior:** The UI must display the `source_title` and a clickable link to the `source_url`.
- **Safety Rule:** The UI must not embed external video players that execute untrusted JavaScript; it must use standard external links.
- **Acceptance Criteria:** Clicking the source link opens the user's default browser to the correct YouTube URL.
- **Source Grounding:** Directly maps to `source_video_id` and `source_url`.

## REQ-005: Show Evidence Snippets
- **Description:** The system must display the raw transcript snippet that generated the prompt.
- **Input Artifacts:** `prompt_library.jsonl`.
- **Output Behavior:** The `evidence_snippet` field must be visible either inline or in a detail modal for every prompt.
- **Safety Rule:** The snippet must be displayed verbatim, without LLM paraphrasing or summarization.
- **Acceptance Criteria:** The user can read the exact words spoken in the video that triggered the prompt.
- **Source Grounding:** Directly maps to `evidence_snippet`.

## REQ-006: Enforce Human Review Required
- **Description:** The system must clearly indicate prompts that require human review.
- **Input Artifacts:** `prompt_library.jsonl`.
- **Output Behavior:** Display a visual indicator for prompts where `human_review_required` is `true`. Display the `suggested_next_human_action` prominently.
- **Safety Rule:** The UI must not allow automated bypass of the human review step.
- **Acceptance Criteria:** A user sees the suggested human action (e.g., "Identify required data before requesting implementation.") next to the prompt.
- **Source Grounding:** Maps to `human_review_required` and `suggested_next_human_action`.

## REQ-007: Prevent Financial Advice / Trading Signal Framing
- **Description:** The UI must explicitly inform the user that it does not provide financial advice.
- **Input Artifacts:** Hardcoded UI text / `WORKFLOW_BROWSER_DESIGN.md`.
- **Output Behavior:** A persistent safety banner must be visible on all screens.
- **Safety Rule:** The banner must state that no financial advice, trading signals, or bot logic is generated.
- **Acceptance Criteria:** The safety banner is visible regardless of which workflow or prompt is selected.
- **Source Grounding:** Aligns with the overall project `active_constraints`.

## REQ-008: Export Selected Prompt to Research Notebook (Future)
- **Description:** The system must allow a user to select a prompt and stage it for export to a Jupyter Notebook template.
- **Input Artifacts:** `prompt_library.jsonl`.
- **Output Behavior:** A "Select" or "Pin" button next to prompts. A "Generate Notebook" action that passes the selected prompt IDs to a future script.
- **Safety Rule:** The exported notebook must contain sterile templates (forcing human-defined in-sample/out-of-sample splits) and must not contain executable trading logic.
- **Acceptance Criteria:** (Design only) The UI mockups/design documents include an export button and a staging area.
- **Source Grounding:** Aligns with the transition from Milestone 3B to 3C.
