# Wireframe Brief Contract

A wireframe brief records UI structure needs for later UI planning. It is not a generated design and not application code.

## Required Sections

- Screens/Pages Required
- User Flows
- Layout Goals
- Key UI States
- Empty/Loading/Error States
- Inputs and Outputs
- Navigation Model
- Accessibility Considerations
- Handoff Notes for Future UI Generation

## Rules

- Use `NOT_SPECIFIED_IN_DISCOVERY_HANDOFF` when the Discovery handoff does not specify wireframe or UI requirements.
- Do not invent screens, flows, or layouts silently.
- Do not generate visual assets or app code.
