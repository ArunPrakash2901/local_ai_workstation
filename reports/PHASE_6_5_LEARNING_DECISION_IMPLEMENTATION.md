# Phase 6.5: Learning Progress Decision Implementation

## Overview
Phase 6.5 implements the `ws learning-decision` command, providing a deterministic classification of learning progress within a stronghold. This command analyzes the latest qualitative assessment and numerical scores to categorize the next safe learning action as `ADVANCE_TO_NEXT_TASK`, `REVIEW_CURRENT_TASK`, or `REPEAT_SESSION`. This ensures the operator follows a structured, evidence-based learning path.

## Files Changed
- **`scripts/ws`**: Integrated `learning-decision` into the help menu and dispatcher. Fixed redundant syntax in the dispatcher tail.
- **`scripts/ws_learning_decision.sh`** (New): Orchestrates the analysis of the latest assessment. It uses Python to parse markdown feedback for explicit recommendations and numeric scores, implementing a state-aware classification matrix.
- **`WORKSTATION_MANUAL.md`**: Updated to document the new `ws learning-decision` command and its role in the validated learning loop.

## Command Behavior
The command `ws learning-decision <id_or_path>` performs the following:
1. **Resolves Stronghold**: Targets `learning` domain workspaces.
2. **Artifact Analysis**: Reads the latest assessment markdown file generated by `ws learning-assess`.
3. **Deterministic Classification**:
   - `ADVANCE_TO_NEXT_TASK`: High score (>= 8) or explicit "ADVANCE" recommendation.
   - `REVIEW_CURRENT_TASK`: Medium score (5-7.9) or explicit "REVIEW" recommendation.
   - `REPEAT_SESSION`: Low score (< 5) or explicit "REPEAT" recommendation.
   - `NEEDS_HUMAN_REVIEW`: Ambiguous or contradictory feedback.
4. **Durable Logging**:
   - Generates a `reports/learning_decision_<timestamp>.md` file inside the stronghold.
   - Appends detailed results to `practice_log.md` and `loop_log.md`.
   - Updates `state.json` with the decision metadata and transitions the session status to `decision_recorded`.

## Limitations
- **Parsing Heuristics**: The classification relies on regex patterns for scores and keywords. Highly unconventional assessment formatting may lead to `NEEDS_HUMAN_REVIEW`.
- **Linear Gating**: The command handles the decision for the current task but does not automatically trigger the generation of the next task's plan.
