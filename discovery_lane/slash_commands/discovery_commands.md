# Discovery Lane Slash Commands

Slash commands are the intended ergonomic surface. In v1.7 they are documented as thin aliases to the executable workstation route:

```text
/discovery approve phase_01_foundation
-> ws discovery approve phase_01_foundation
-> python discovery_lane/tools/discovery_command.py approve phase_01_foundation
```

`ws discovery ...` is the current executable workstation interface. The Python dispatcher remains the local fallback.

The executable fallback is:

```powershell
python discovery_lane/tools/discovery_command.py <command> [args]
```

## Commands

| Slash Command | Status | Purpose | Underlying Action | Safety Behavior |
| --- | --- | --- | --- | --- |
| `/discovery intake-set <path>` | IMPLEMENTED | Validate a set of already-created phase-wise Markdown research reports before ingest. | `ws discovery intake-set <path>` | Writes set-level manifest and intake report artifacts only. No phase packet generation, model calls, execution, or git actions. |
| `/discovery ingest` | IMPLEMENTED | Convert Markdown research reports into phase packets, worker prompts, manifests, and index entries. | `ws discovery ingest` | Writes only Discovery Lane generated artifacts. No model calls or execution. |
| `/discovery ingest-set <set_id>` | IMPLEMENTED | Convert one `READY_FOR_INGEST` research set into set-prefixed phase packets and worker prompts. | `ws discovery ingest-set <set_id>` | Writes derived Discovery Lane phase artifacts for that set only. No approval, handoff, branch, execution, model call, commit, push, or merge. |
| `/discovery approve-set <set_id> --dry-run` | IMPLEMENTED | Plan human approval review for packets generated by one ingested research set. | `ws discovery approve-set <set_id> --dry-run` | Read-only unless `--write-report` is explicitly used to write an advisory Markdown review plan. No approval, handoff, branch plan, execution, model call, commit, push, or merge. |
| `/discovery handoff-list` | IMPLEMENTED | List approved handoff bundles and planned branch metadata. | `ws discovery handoff-list` | Read-only. No worker execution, branch creation, approval, commit, push, or merge. |
| `/discovery queue-plan <set_id>` | IMPLEMENTED | Build a non-executing execution queue manifest for individually approved handoffs in one research set. | `ws discovery queue-plan <set_id>` | Writes only queue planning JSON. No execution, branch creation, checkout, approval, handoff creation, commit, push, or merge. |
| `/discovery queue-plan <set_id> --write-report` | IMPLEMENTED | Build a non-executing execution queue manifest and Markdown report. | `ws discovery queue-plan <set_id> --write-report` | Writes only queue planning artifacts. No execution, branch creation, checkout, approval, handoff creation, commit, push, or merge. |
| `/discovery status` | IMPLEMENTED | Show Discovery Lane counts and index. | `ws discovery status` | Read-only. |
| `/discovery review-list` | IMPLEMENTED | List packets that need human review. | `ws discovery review-list` | Read-only. |
| `/discovery approve <phase_or_packet_id>` | IMPLEMENTED | Record human approval and create handoff, branch plan, and approval record. | `ws discovery approve <phase_or_packet_id>` | Writes Discovery Lane approval artifacts only. No git actions. |
| `/discovery approve <phase_or_packet_id> --branch <branch>` | IMPLEMENTED | Approve with a custom planned branch name. | `ws discovery approve <phase_or_packet_id> --branch <branch>` | Records branch plan only. Does not create a branch. |
| `/discovery reject <phase_or_packet_id> --reason "..."` | IMPLEMENTED | Record human rejection after external review. | `ws discovery reject <phase_or_packet_id> --reason "..."` | Writes rejection record and packet snapshot only. |
| `/discovery handoff <phase_or_packet_id>` | IMPLEMENTED | Show an approved handoff bundle. | `ws discovery handoff <phase_or_packet_id>` | Read-only. |
| `/discovery branch-plan <phase_or_packet_id>` | IMPLEMENTED | Show the planned branch metadata. | `ws discovery branch-plan <phase_or_packet_id>` | Read-only. Does not run git. |
| `/discovery audit` | IMPLEMENTED | Audit Discovery Lane state and safety boundaries. | `ws discovery audit` | Read-only. |
| `/discovery help` | IMPLEMENTED | Show local Discovery Lane command help. | `ws discovery help` | Read-only. |

## Examples

```text
/discovery intake-set discovery_lane/inbox/product_idea_alpha
/discovery ingest-set product_idea_alpha
/discovery approve-set product_idea_alpha --dry-run
/discovery handoff-list
/discovery queue-plan product_idea_alpha --write-report
/discovery ingest
/discovery review-list
/discovery approve phase_01_foundation --branch work/discovery/phase-01-foundation
/discovery reject phase_01_foundation --reason "Acceptance criteria are incomplete."
/discovery audit
/discovery status
```

Raw fallback equivalents:

```powershell
python discovery_lane/tools/intake_research_set.py --input discovery_lane/inbox/product_idea_alpha --output discovery_lane
python discovery_lane/tools/discovery_command.py ingest-set product_idea_alpha
python discovery_lane/tools/discovery_command.py approve-set product_idea_alpha --dry-run
python discovery_lane/tools/discovery_command.py handoff-list
python discovery_lane/tools/discovery_command.py queue-plan product_idea_alpha --write-report
python discovery_lane/tools/discovery_command.py ingest
python discovery_lane/tools/discovery_command.py review-list
python discovery_lane/tools/discovery_command.py approve phase_01_foundation --branch work/discovery/phase-01-foundation
python discovery_lane/tools/discovery_command.py reject phase_01_foundation --reason "Acceptance criteria are incomplete."
python discovery_lane/tools/discovery_command.py audit
python discovery_lane/tools/discovery_command.py status
```

## Boundary

Slash command approval does not execute phase work. It does not create, checkout, merge, push, or delete git branches. It only records that a human-approved packet may be used as an input by a later execution lane.
